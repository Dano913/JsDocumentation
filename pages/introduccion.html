Introducción

Como se lee el código JavaScript

JavaScript interpreta el código línea por línea, de arriba hacia abajo y de izquierda a derecha. El motor de JavaScript analiza cada línea y ejecuta las instrucciones que encuentra.

Veamos un paso a paso a "cámara lenta" de cómo es este proceso en cuestión de milisegundos.

Análisis léxico

El motor de JavaScript comienza con un análisis léxico, donde divide el código en unidades más pequeñas llamadas "tokens".

Estos tokens pueden ser palabras clave, identificadores, literales, operadores y símbolos.

Análisis sintáctico

Luego, el motor de JavaScript realiza un análisis sintáctico para determinar la estructura del código.

Se verifica que la sintaxis sea correcta y que las estructuras como las funciones, los bucles y las condiciones estén correctamente formadas.

Análisis semántico

En el análisis semántico, el motor de JavaScript busca el significado de los tokens y las estructuras.

Se verifican los tipos de datos, las referencias a variables y funciones, y se asigna memoria para las variables y objetos.

Ejecución

Finalmente, el motor de JavaScript ejecuta las instrucciones del código.

Las operaciones matemáticas se realizan, las funciones se llaman, las condiciones se evalúan y los bucles se ejecutan.

¿Qué es una variable?

La variable es el nombre de la caja en la que podemos guardar nuestra información, pueden contener productos, usuarios, y cualquier otro dato. Para crear una variable en Javascript usamos la palabra let. Después indicaremos el nombre de nuestra variable: letproduct;.

Ahora podremos usar esta variable y asignarle un valor con el operador de asignación =. Es como si estuviéramos metiendo la palabra "Zumito" dentro de la caja llamada product


            let product;
            product = 'Zumito';
            console.log(product);

Declarar y asignar

Podemos ser más directos a la hora de declarar una variable y hacerlo en una única línea.


            let product = 'Zumito';
            console.log(product);

Cambiar y reasignar el valor en variables

Podemos seguir cambiando el valor de nuestras variables conforme pasamos de linea, vamos a ver como nuestros "products" cambian:


            let product;
            console.log(product);
            // Undefined

            product = "Zumito";
            console.log(product);
            // Zumito

            product = "Patata";
            console.log(product);
            // Patata

Además de cambiar el valor podemos copiar el valor de una variable a otra:


            let product = 'Tomate';
            let myProduct;
            myProduct = product;
            
            console.log(myProduct);
            // Tomate
            
            console.log(product);
            // Tomate

Declaración única

Las variables solo deben declararse una vez, repetir la declaración de una variable disparará un error de sintaxis:


            let product = 'Tomate';
            let product = 'Patata';
            // SyntaxError: 'product' has already been declared

¿Qué nombre le asignamos a nuestras variables?

A la hora de asignar un nombre a nuestras variables seguiremos dos normas:

  • El nombre de la variable solo debe contener letras, dígitos y los símbolos $ y _ (guión bajo).
  • El primer carácter no debe ser un dígito.

Consejo: el nombre de las variables debería ser un nombre descriptivo del valor que puede contener, si vamos a guardar usuarios el nombre apropiado sería users.

Nombres no válidos o reservados

Nuestras variables tampoco pueden llamarse como las palabras que utiliza el propio lenguaje para definir las diferentes funcionalidades.

Estas palabras se llaman "palabras reservadas" y las iréis conociendo a lo largo de JavaScript.

            let my-product;
            // No se pueden utilizar guiones en los nombres
            
            let let = 'Hola';
            // No puedes llamar let a una variable
            
            let return = 'Hola';
            // No puedes llamar return a una variable

Expresiones y declaraciones

En cuanto a las expresiones y declaraciones matemáticas que podemos utilizar os indicamos las siguientes:

  • El operador + como os podéis imaginar nos servirá para sumar dos expresiones siempre y cuando estemos hablando de números, en cuanto a strings el operador + servirá para concatenar dos cadenas de texto
  • El operador - nos servirá para restar dos expresiones numéricas
  • El operador * nos servirá para multiplicar dos expresiones numéricas
  • El operador / nos servirá para dividir dos expresiones numéricas y obtener el resultado de la división
  • El operador % nos servirá para dividir dos expresiones numéricas y obtener el resto de la división

Scope

El término scope se refiere al contexto o entorno en el que se declaran las variables y determina su accesibilidad. En otras palabras, define dónde puedes usar una variable específica dentro de tu código.

  • Scope global: Las variables declaradas fuera de cualquier función tienen alcance global. Son accesibles desde cualquier lugar del código. Sin embargo, se recomienda usar el scope global con moderación para evitar conflictos de nombres y problemas de organización del código.
  • Scope local: Las variables declaradas dentro de un bloque tienen alcance local. Solo son accesibles desde dentro de ese bloque y no son visibles desde fuera. Esto ayuda a mantener el código organizado y evita que las variables se sobrescriban accidentalmente.

            // Ejemplo 1
            if (5 > 4) {
                let color = 'rosa';
            }
            console.log(color); 
            // undefined -> NO ES ACCESIBLE DESDE FUERA

            // Ejemplo 2
            if (5 > 4) {
                let color = 'rosa';
                console.log(color); 
                // rosa -> ES ACCESIBLE PORQUE ESTÁ DENTRO DE SU SCOPE
            }

            // Ejemplo 3
            let color = 'rosa';
            if (5 > 4) {
                let color = 'rosa';
                console.log(color);
                // rosa -> ES ACCESIBLE PORQUE COLOR ESTÁ A NIVEL GLOBAL
            }

Como ejemplo podemos imaginar que en nuestro domicilio si aplaudimos nadie en la calle nos escucha, pero si alguien aplaude en la acera de nuestro edificio si que lo escuchamos desde casa.

De fuera hacia dentro es accesible pero de dentro (privado) hacia afuera no es tan sencillo.

¿Qué es un bloque de código?

Un bloque es una declaración que contiene líneas de código que se ejecutan de manera diferente al flujo normal del código, hay bloques condicionales, bucles, funciones, etc.

Estos bloques estan siempre envueltos entre llaves (no son objetos, no los confundáis).


            if (5 < 4) {
                // principio del código del bloque {
                // final del código del bloque }
            }

Variables locales

Una variable declarada dentro de un bloque solo es visible dentro de ese bloque.

Dicho de otro modo el bloque genera un scope propio.


            function showPokemon() {
                let pokemon = "Pikachu"; // variable local
                console.log(pokemon);
            }
            showPokemon(); // Pikachu
            console.log(pokemon); // <-- Error! La variable es local para esta función

Variables externas o globales

Un bloque si puede acceder a una variable externa.

Puedes acceder a variables globales dentro del scope de un bloque.


            let pokemonExt = 'Charmander';

            function showPokemon() {
            console.log(pokemonExt);
            }

            showPokemon(); // Charmander

Shadowing

Si una variable con el mismo nombre se declara dentro de la función, le hace sombra [shadowing] a la externa.

Por ejemplo, en el siguiente código, la función usa el pokemonExt local.

El exterior se ignora, es decir, prevalece la local.


            let pokemonExt = 'Charmander';
            function showPokemon() {
            let pokemonExt = "Squirtle"; // declara variable local

            console.log( pokemonExt ); // Squirtle
            }

            // la función crea y utiliza su propia variable local pokemonExt
            showPokemon();

            console.log( pokemonExt );
            // Charmander   ->   se mantiene, la función no accedió a la variable externa porque estádentro
            // tiene una declaración de una variable que se llama igual y por lo tanto no afecta a la variable global
            // si no a su variable local.

Tipos de datos

Aunque JavaScript no es un lenguaje fuertemente tipado, esto no quiere decir que no existan tipos de datos. Vamos a ver los tipos que existen en JavaScript:

  • String: Nos permite almacenar cadenas de texto
  • Number: Nos permite almacenar valores numéricos(tanto enteros como con decimales).
  • Boolean: Nos permite almacenar valores de verdadero o falso(true ó false).
  • Null: Nos permite almacenar un valor nulo o vacío.
  • Undefined: Nos permite almacenar una variable que ha sido declarada pero no inicializada.
  • Object: Nos permite almacenar colecciones de datos y entidades más complejas
  • Array: Nos permite almacenar listas de valores

Tipado débil

JavaScript es un lenguaje dinámico o de tipado débil, ya que no es obligatorio indicar el tipo de una variable al definirlo.

De hecho, una misma variable puede cambiar de tipo sin problema alguno durante el proceso de ejecución de nuestro código.

Podemos realizar pruebas haciendo uso de la función typeof, la cual nos indicará el tipo de una variable en cada momento, os recomendamos probarlo vosotros mismos.


            let txt = typeof "texto"; // string
            let num = typeof 10;     // number
            let bool = typeof true; // boolean

Datos dinámicos

JavaScript tiene tipos de datos dinámicos, lo que significa que la comprobación de tipo se realiza en tiempo de ejecución en lugar de tiempo de compilación. Con los lenguajes de tipo dinámico, se puede utilizar una variable del mismo nombre para contener diferentes tipos de datos.

Por ejemplo, la variable t, definida como una variable por la palabra clave let, se puede asignar para contener diferentes tipos de datos o puede inicializarse pero no se define.


            let tipo;
            // tipo es undefined;

            tipo = 6;
            // tipo es number;

            tipo = true;
            // tipo es boolean;

            tipo = "cadena";
            // tipo es string;

Operadores lógicos

Los operadores lógicos son un tipo de operadores en JavaScript que se utilizan para evaluar expresiones lógicas y devolver un valor booleano (true o false).

Los operadores lógicos más comunes son && (and), || (or) y ! (not).

Operadores && (AND)


            // expresión1 && expresión2

            // ejemplos:
            console.log(true && true); // true
            console.log(true && false); // false
            console.log(false && true); // false
            console.log(false && false); // false

            // ejemplo con varias expresiones AND concatenadas, siempre y cuando haya una false ya será false.
            console.log(true && true && true && false && true); // false

Operadores || (OR)

El operador || evalúa dos expresiones lógicas y devuelve true si al menos una de ellas es true, y false sólo si ambas son false.

La sintaxis del operador || es la siguiente:


            // expresión1 || expresión2

            // ejemplos:
            console.log(true || true); // true
            console.log(true || false); // true
            console.log(false || true); // true
            console.log(false || false); // false

            // ejemplo con varias expresiones OR concatenadas, siempre y cuando haya una true ya será true.
            console.log(false || false || true || false || false); // true

Operador ! (NOT)

El operador ! invierte el valor booleano de su operando. Es decir, si el operando es true, el operador ! lo convierte en false, y viceversa.

La sintaxis del operador ! es la siguiente:


            // !operando

            // ejemplos:
            console.log(!true); // false
            console.log(!false); // true

            // cuando concatenamos varios operadores NOT seguidos no obtenemos ningún error,
            // tenemos que leer de derecha a izquierda como van negando lo anterior,
            // en este caso al ser pares será false, si fuesen impares sería true,
            // esto no es tan común verlo pero es para que veáis que no hay ningún fallo como tal y
            // simplemente cada ! niega lo anterior de derecha a izquierda
            console.log(!!!!false); // false

Operadores en conjunto

Obviamente podremos utilizar estos operadores para que trabajen en conjunto, al trabajar en conjunto podremos ver cómo los diferentes operadores lógicos tienen orden de jerarquía, el && tiene prioridad ante el ||, sería lo mismo que una multiplicación frente a una suma en una ecuación de segundo grado.


            // paso 1:
            // true && true = true
            console.log(true && true || false || true && true && false); // true

            // simplificamos:
            // paso 2:
            // true && true && false = false
            console.log(true || false || true && true && false);

            
            // simplificamos:
            // paso 3:
            // true || false || false = true
            console.log(true || false || false);

Operadores de comparación

Es importante que podamos ser capaces de comparar valores, por ejemplo, si el destino que estás buscando es igual al nombre de un destino almacenado en mi aplicación te lo muestro, si la edad de usuario es menor de 18 no te dejo registrarte, etc.

Para ello tenemos los siguientes comparadores que os resultarán muy familiares:

  • < Menor que
  • <= Menor o igual que
  • > Mayor que
  • >= Mayor o igual que
  • == Igual que
  • === Estrictamente igual que
  • != Diferente que
  • !== Estrictamente diferente que
